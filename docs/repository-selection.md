## Selection criterion

A repository belongs in the main path when a reviewer can see:

- what it does;
- why it matters;
- what is inspectable;
- current maturity level;
- how it connects to self-maintaining agentic R&D systems.

A surface belongs in the main path when it shows behavior a repository alone cannot show:

- live state;
- public/private lens separation;
- Lab cycle state;
- dashboard evidence;
- operational continuity;
- transfer from domain + intent into a Lab workflow.

## Boundary

The THIA implementation repository is private.

THIA as a system is not private. It has public surfaces and should be explained through architecture, screenshots, public behavior and curated review material.
