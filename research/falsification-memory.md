# Falsification Memory

The cimitero/falsification memory is one of the strongest parts of the D-ND
workflow. It shows not only what the system claims, but what it rejected and why.

## Public Framing

Use this as:

```text
Negative results and rejected claims
```

not as a raw dump of internal lab memory.

## What to Include

- claim or hypothesis;
- evidence used to test it;
- falsifier or counter-pole result;
- verdict;
- what changed in the next cycle.

## Why It Matters

This is the operational answer to a common AI failure: repeated confident
mistakes. A system that keeps a memory of rejected paths can avoid rediscovering
the same false claim after context loss.
